# 小红书发帖指南

**重要说明**：请严格按照以下结构和规则生成小红书笔记内容。所有生成内容必须基于真实信息，避免夸大或虚构。

**最新要求**：
- **内容长度**：控制在800-1200字，避免超过1500字
- **表情符号**：每篇文章使用不超过5个emoji，保持克制
- **语气风格**：真诚有人情味，避免过于营销化或机械化表达

---

## 🎭 人设获取

**重要**：在生成内容前，必须先调用 `xhs_get_my_profile` 工具获取当前登录用户的资料信息，包括：
- 账户名称
- 粉丝数量
- 关注数量
- 获赞与收藏数
- 小红书账号ID
- 个人描述
- 账户状态

**人设使用原则**：
1. **基于真实资料**：根据获取的用户资料信息，确定内容的人设和身份定位
2. **语气与风格**：
   - **专业但亲切**：像朋友分享经验，而不是销售推销
   - **真实可信**：使用第一人称，分享真实经历和个人感悟
   - **真诚有人情味**：避免模板化语言，加入个人思考和情感
   - **价值导向**：每篇文章必须提供可操作的方法或工具，真正帮助用户解决问题
   - **互动性强**：结尾设置开放性问题，引导真实讨论
3. **内容适配**：根据用户的专业领域、粉丝规模、账户定位等信息，调整内容的专业深度和表达方式

## ✏️ 标题生成规则

### 字数要求
- **12-20个中文字符**（约60-100字符）
- 英文单词按1个字符计算
- **避免过短（<10字）或过长（>25字）**

### 标题语气要求
- **真诚自然**：像朋友聊天，而不是广告标题
- **问题导向**：引发读者思考和共鸣
- **价值明确**：让读者一看就知道能学到什么

### 标题模板

**通用模板**：
```
[真实疑问/个人经历] + [具体方法/解决方案] + [实际效果]
[真实问题] + [解决方案] + [使用体验]
[身份经历] + [真实挑战] + [心得感悟]
[个人状态] + [方法转变] + [具体内容]
```

**示例**：
```
"为什么我总是记不住？3个科学记忆方法"
"学习效率低？试试间隔重复技巧"
"多端同步总出问题？我这样解决了"
"从混乱笔记到知识体系，只需3步"
"独立开发一年，我踩过的5个坑"
"从拖延到高效，我用了3个技巧"
```

### 标题关键词（必须包含2-3个）
- 学习、效率、方法、工具、笔记、记忆
- 为什么、如何、3个、5个、实用、经验
- 免费、简单、真实、有效、快速

### 避免的标题模式
- ❌ 纯产品名："XX新功能介绍"（避免直接产品名）
- ❌ 夸大其词："学习神器，效率100倍"
- ❌ 过于泛化："学习技巧分享"
- ❌ 命令式："必须试试这个方法"

## 📄 内容格式要求

### 严格字数控制
- **总字数**：800-1200字（**不超过1200字**）
- **每段长度**：80-150字（3-5行）
- **小标题频率**：每300-400字一个
- **配图位置**：每2-3段后插入一张

### 段落写作原则
1. **第一人称视角**：用"我"、"我们"、"我的经验"，增加真实感
2. **故事化表达**：用具体场景描述，避免抽象说教
3. **问题引导**：每段开头可提一个小问题，引出内容
4. **自然过渡**：用"说到这里"、"其实还有"、"让我分享一个例子"连接段落

### 表情符号使用规范
- **总数限制**：每篇文章**不超过5个emoji**
- **位置要求**：
  - 标题：0-1个（可选）
  - 小标题：0-1个/标题
  - 正文：每500字1个，避免连续使用
- **类型推荐**：
  - 📚 学习相关（最多1次）
  - 💡 思考洞察（最多1次）
  - ⚡ 效率工具（最多1次）
  - 👇 互动引导（结尾使用）

**避免**：过度使用😊👍❤️等通用表情，保持专业感。

## 🖼️ 配图规范

### 图片数量要求
- **基础要求**：3-4张（**不要超过5张**）
- **最少数量**：至少2张，确保内容有视觉支撑
- **最佳数量**：3-4张，平衡信息量和阅读体验

### 图片类型与用途

**类型1：内容展示图（1-2张，推荐）**
- **用途**：展示文章核心内容、方法步骤、数据图表等
- **要求**：
  - 清晰展示关键信息
  - 文字清晰可读，字体大小适中
  - 配色协调，视觉舒适
  - 避免信息过载，重点突出
- **位置**：核心内容段落中
- **说明**：在正文中标注"[配图X：XXX展示]"

**类型2：示例/案例图（1-2张，推荐）**
- **用途**：展示实际应用场景、使用效果、前后对比等
- **要求**：
  - 真实自然，避免过度美化
  - 与正文案例直接对应
  - 体现实际使用场景
  - 避免隐私信息泄露
- **位置**：案例分享或效果展示部分
- **说明**：在正文中标注"[配图X：XXX案例]"

**类型3：辅助说明图（0-1张，可选）**
- **用途**：流程图、思维导图、原理示意图等
- **要求**：
  - 简洁明了，逻辑清晰
  - 标注关键节点和数据
  - 与正文内容直接对应
  - 颜色搭配协调
- **位置**：原理或方法说明部分
- **说明**：在正文中标注"[配图X：XXX原理图]"

### 图片格式规范

**尺寸要求**：
- **宽度**：1080px（小红书标准宽度）
- **比例**：1:1（正方形）或 4:5（竖版，推荐）
- **最小尺寸**：宽度不低于720px

**质量要求**：
- **清晰度**：高清无水印，文字清晰可读
- **格式**：JPG、PNG、WebP（推荐JPG，文件小）
- **文件大小**：单张不超过5MB，建议控制在2MB以内
- **压缩**：保持清晰度的前提下适当压缩

**内容要求**：
- **文字可读性**：图片中的文字必须清晰可读
- **信息完整性**：关键信息完整展示，不被裁剪
- **隐私保护**：避免包含个人信息、账号信息等敏感内容
- **版权合规**：使用原创图片或已授权图片，避免侵权

### 图片获取方式

**方式1：本地图片（推荐）**
- 将图片文件放在缓存目录：`~/.xhs-cli/post/images/`
- 在笔记中使用相对路径引用，如：`image1.jpg`
- 支持常见格式：jpg、png、gif、webp

**方式2：网络图片（支持）**
- 直接使用图片URL，如：`https://example.com/image.jpg`
- 系统会自动下载到缓存目录
- 支持 http:// 和 https:// 协议

**方式3：生成图片**
- 可以生成图表、流程图等辅助图片
- 保存到缓存目录后使用

### 图片在正文中的使用

**标注格式**：
- 在需要插入图片的位置标注：`[配图X：图片说明]`
- 例如：`[配图1：方法步骤演示]`、`[配图2：使用效果对比]`
- 图片说明要简洁明了，与内容对应

**插入位置**：
- 每2-3段后插入一张图片
- 图片要紧跟相关文字内容
- 避免图片集中在开头或结尾
- 第一张图片建议在开头100-200字后

**图片顺序**：
- 按照在正文中出现的顺序编号
- 配图1、配图2、配图3...依次排列
- 确保图片顺序与标注顺序一致
