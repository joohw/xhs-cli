一次手工发笔记不难，难的是**稳定、可复现、可批量**的发布流程——尤其当文案由 AI 生成、图片由设计工具导出、运营同学只负责「点发布」时。

## 推荐工作流

1. **准备素材** — 图片、标题、正文、话题标签；建议目录结构固定，便于脚本读取。
2. **登录态检查** — 确保 xhs-cli 已完成创作者账号登录。
3. **执行发布命令** — 按 README 调用 CLI 发布能力（参数以当前版本为准）。
4. **验证结果** — 在创作者中心确认笔记状态，记录链接或 ID 到表格/数据库。

```bash
npm install -g xhs-cli
# 具体发布命令见 GitHub README
```

## 与 Agent 配合

常见分工：

- **Agent** — 根据选题生成标题、正文、标签建议
- **xhs-cli** — 在平台侧完成上传与发布
- **人** — 审核敏感内容与品牌调性后再执行

这样 Agent 不需要直接操作浏览器 DOM，CLI 提供稳定边界。

## 注意事项

- 遵守小红书平台规范与频率限制
- 发布前人工抽检 AI 生成内容
- 账号 Cookie / 登录态妥善保管，勿泄露
