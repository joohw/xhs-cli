# 小红书发帖指南

**重要说明**：请严格按照以下结构和规则生成小红书笔记内容。所有生成内容必须基于真实信息，避免夸大或虚构。

**最新要求**：
- **内容长度**：控制在800-1200字，避免超过1500字
- **表情符号**：每篇文章使用不超过5个emoji，保持克制
- **语气风格**：真诚有人情味，避免过于营销化或机械化表达

---

## 📚 写作前准备

### 1. 查看范文（必须）

**重要**：在开始写作前，必须先查看范文参考。通过资源查看范文
- 查看 MCP 资源列表中的 `xhs://examples/` 资源，选择相关范文进行参考
- 直接读取资源 URI：`xhs://examples/{filename}`


**范文使用原则**：
- 参考范文的结构、语气和表达方式
- 学习范文的段落划分和过渡技巧
- 借鉴范文的互动方式和结尾设计
- **不要直接复制**，要根据当前主题进行原创写作

### 2. 获取人设信息（必须）

**重要**：在生成内容前，必须先调用 `xhs_get_my_profile` 工具获取当前登录用户的资料信息，包括：
- 账户名称
- 粉丝数量
- 关注数量
- 获赞与收藏数
- 小红书账号ID
- 个人描述
- 账户状态

**人设使用原则**：
1. **基于真实资料**：根据获取的用户资料信息，确定内容的人设和身份定位
2. **语气与风格**：
   - **专业但亲切**：像朋友分享经验，而不是销售推销
   - **真实可信**：使用第一人称，分享真实经历和个人感悟
   - **真诚有人情味**：避免模板化语言，加入个人思考和情感
   - **价值导向**：每篇文章必须提供可操作的方法或工具，真正帮助用户解决问题
   - **互动性强**：结尾设置开放性问题，引导真实讨论
3. **内容适配**：根据用户的专业领域、粉丝规模、账户定位等信息，调整内容的专业深度和表达方式

## 标题规范

### 字数要求
- **12-20个中文字符**（约60-100字符）
- 英文单词按1个字符计算
- **避免过短（<10字）或过长（>25字）**

### 标题语气要求
- **真诚自然**：像朋友聊天，而不是广告标题
- **问题导向**：引发读者思考和共鸣
- **价值明确**：让读者一看就知道能学到什么

### 标题模板

**通用模板**：
```
[真实疑问/个人经历] + [具体方法/解决方案] + [实际效果]
[真实问题] + [解决方案] + [使用体验]
[身份经历] + [真实挑战] + [心得感悟]
[个人状态] + [方法转变] + [具体内容]
```

**示例**：
```
"为什么我总是记不住？3个科学记忆方法"
"学习效率低？试试间隔重复技巧"
"多端同步总出问题？我这样解决了"
"从混乱笔记到知识体系，只需3步"
"独立开发一年，我踩过的5个坑"
"从拖延到高效，我用了3个技巧"
```




## 正文规范

### 字数控制
- **总字数**：800-1200字（**不超过1200字**）
- **每段长度**：80-150字（3-5行）
- **小标题频率**：每300-400字一个

### 正文结构要求

**基本结构**：
1. **开头（100-150字）**：引入话题，引发共鸣
   - 用真实场景或问题开头
   - 快速建立与读者的连接
   - 明确文章要解决的问题

2. **主体（600-900字）**：核心内容，分3-5个段落
   - 每个段落围绕一个核心点展开
   - 段落之间要有逻辑递进关系
   - 用具体案例、方法、工具支撑观点

3. **结尾（100-150字）**：总结升华，引导互动
   - 总结核心要点
   - 设置开放性问题
   - 鼓励读者分享经验

### 段落写作原则

1. **第一人称视角**：用"我"、"我们"、"我的经验"，增加真实感
2. **故事化表达**：用具体场景描述，避免抽象说教
3. **问题引导**：每段开头可提一个小问题，引出内容
4. **自然过渡**：用"说到这里"、"其实还有"、"让我分享一个例子"连接段落


### 分段要求

**流畅自然的分段原则**：
- **逻辑清晰**：每段只讲一个核心观点，避免内容混杂"
- **节奏适中**：长短段落交替，避免所有段落长度一致
- **视觉友好**：每段3-5行，避免大段文字造成阅读疲劳


### 图文混排说明

**重要**：小红书不支持图文混排功能。

**注意事项**：
- **不要**在正文中插入图片标记或占位符（如 `[图片1]`、`[配图]` 等）
- **不要**在正文中描述图片内容或位置
- **不要**使用"上图"、"下图"、"配图"等指向图片的词汇
- 图片由系统单独上传，与正文内容分离
- 正文只需专注于文字内容的流畅性和完整性

**正确的做法**：
- 正文内容完整独立，不依赖图片说明
- 如果内容需要配图，在生成内容后单独处理图片上传
- 正文中可以用文字描述场景，但不要提及图片
